# 3.2.reflect.test.defects

## classifications

### defect 1: typescript version mismatch

**classification**: repo quirk

**citation**: dependencies has `"typescript": "^3.8.3"` (legacy from 2020) while devDependencies has `"typescript": "5.4.5"`. pnpm resolved to 3.x because that's what the production dependency says. the real issue is the outdated constraint in dependencies, not pnpm behavior.

**action**: update dependencies typescript to `"^5.0.0"` since the code works with typescript 5.x. pnpm.overrides was a temporary workaround, not a permanent fix.

### defect 2: environment utility not found

**classification**: practice bug

**citation**: declapract upgrade added `blackbox/environment.ts` which imports from `src/utils/environment.ts`, but this file did not exist. the upgrade should either: (a) create the dependency, or (b) not create files that depend on absent modules.

**action**: file bug in declapract-typescript-ehmpathy about blackbox/environment.ts dependency chain.

### defect 3: circular dependencies in source

**classification**: repo quirk

**citation**: guard files used unusual import path `@src/domain.operations/..` which expanded to parent directory. this was a pre-extant code pattern in this repo.

**action**: no infrastructure change needed; fixed locally.

### defect 4: dpdm reported node_modules cycles

**classification**: practice bug (update)

**citation**: originally documented as "add node_modules to exclude", but this is forbidden per rule.forbid.dpdm-exclude-change.

**actual fix**: replaced joi with zod to eliminate cycles at source. upgraded domain-objects and helpful-errors to versions without cycles. the dependency cycles in joi, domain-objects, and helpful-errors were upstream issues that have since been fixed or avoided.

**action**: track upstream dependency cycle fixes; prefer zod over joi for schema validation.

### defect 5: depcheck flagged unused dependencies

**classification**: repo quirk

**citation**: flagged packages are indirect dependencies used by tooling (test-fns, ts-jest) or infrastructure (rhachet-roles). this is expected in tooling-heavy repos.

**action**: no infrastructure change needed; added to .depcheckrc.yml ignores locally.

### defect 6: keyrack extends absent role

**classification**: practice bug

**citation**: `.agent/keyrack.yml` was created with reference to `repo=bhuild/role=dispatcher` which does not exist in this project. the upgrade should validate that extended roles exist.

**action**: file bug in declapract-typescript-ehmpathy about keyrack.yml validation.

### defect 7: format issues

**classification**: expected behavior

**citation**: upgrades modify files; format discrepancies are expected and fixed by standard `npm run fix:format`.

**action**: no change needed; standard workflow.
