## critical hazards

### BLOCKER: domain exports broken

**severity**: BLOCKER ‚Äî build will fail

**what changed**:
- old: `src/domain/index.ts` re-exported from `./constants` AND `./objects`
- old: `src/domain/objects/index.ts` exported DomainObjectMetadata, DomainObjectReferenceMetadata, DomainObjectPropertyMetadata
- new: `src/domain.objects/index.ts` ONLY exports from `./constants` (domain objects NOT re-exported)
- new: `src/index.ts` still imports from `./domain` which no longer exists

**why it breaks**:
- `src/index.ts` imports from `./domain` but folder is now `./domain.objects`
- even if path fixed, `domain.objects/index.ts` only exports constants, not the domain objects

**fix required**:
1. update `src/domain.objects/index.ts` to export domain objects:
   ```ts
   export * from './constants';
   export * from './DomainObjectMetadata';
   export * from './DomainObjectReferenceMetadata';
   export * from './DomainObjectPropertyMetadata';
   ```
2. update `src/index.ts` to import from `./domain.objects` instead of `./domain`

### BLOCKER: typescript version incompatibility

**severity**: BLOCKER ‚Äî build will fail

**what changed**:
- biome format added `import { type ... }` syntax (TypeScript 4.5+ feature)
- package.json has `"typescript": "^3.8.3"` in dependencies
- pnpm installed typescript 3.9.10 (satisfies ^3.8.3 but doesn't support type-only imports)

**why it breaks**:
- `import { type ClassDeclaration, ... }` causes "TS1005: ',' expected" errors
- TypeScript 3.9 doesn't understand the `type` keyword in import statements

**fix required**:
remove `type` keyword from imports, or update typescript dependency to 5.x
