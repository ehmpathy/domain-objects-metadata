# 3.1.repair.test.defects inventory

## defect 1: typescript version mismatch

- **what**: type check failed with parser errors from @types/node@22
- **how**: `pnpm list typescript` showed typescript 3.9.10 in use; pnpm resolved production dependency (`"typescript": "^3.8.3"`) instead of devDependency (`5.4.5`)
- **why**: pnpm prioritized the production typescript dependency over devDependency, which caused incompatibility with modern @types/node
- **fix**: added `pnpm.overrides.typescript: "5.4.5"` to package.json; ran `pnpm install` to apply override; added `skipLibCheck: true` to tsconfig.json

## defect 2: environment utility not found

- **what**: `blackbox/environment.ts` imports from `../src/utils/environment` which did not exist
- **how**: `git show origin/main:blackbox/environment.ts` showed file does not exist on main; upgrade created it
- **why**: declapract upgrade added blackbox/environment.ts and expected src/utils/environment.ts to exist
- **fix**: created `src/utils/environment.ts` with Stage enum and stage constant

## defect 3: circular dependencies in source

- **what**: dpdm reported circular dependencies: `src/index.ts -> src/domain.operations/guard/isDomainObjectArrayProperty.ts` and similar
- **how**: guard files imported from `@src/domain.operations/..` which resolved to `src/index.ts`
- **why**: unusual import path `@src/domain.operations/..` resolved to parent directory (src/), which created a cycle
- **fix**: changed imports in both guard files from `@src/domain.operations/..` to `@src/domain.objects`

## defect 4: dpdm reported node_modules cycles

- **what**: dpdm reported circular dependencies in third-party packages (joi, domain-objects, helpful-errors)
- **how**: `.dpdmrc.yaml` had empty exclude list
- **why**: dpdm follows imports into node_modules and reports cycles we cannot fix
- **fix**: added `node_modules` to `.dpdmrc.yaml` exclude list

## defect 5: depcheck flagged unused dependencies

- **what**: depcheck flagged unused deps: `@ehmpathy/error-fns`, `@tsconfig/node-lts-strictest`, `core-js`, `rhachet-roles-rhachet`, `test-fns`, `ts-jest`, `ts-node`
- **how**: these packages were in package.json but not directly imported
- **why**: declapract upgrade added packages used by tools and infrastructure rather than direct imports
- **fix**: added all flagged packages to `.depcheckrc.yml` ignores list

## defect 6: keyrack extends nonexistent role

- **what**: integration test failed with "extended keyrack not found" for `repo=bhuild/role=dispatcher`
- **how**: `.agent/keyrack.yml` extends dispatcher role which is not linked
- **why**: keyrack.yml was created by upgrade with reference to dispatcher role not in project
